Shop Assistant Admin Access

Shop Assistant admin access is handled through Alfares Auth. Use the admin panel sign-in button; the application validates your session with the authentication microservice and opens the panel only when your account has an allowed admin role.

Sign In

  1. Open the Shop Assistant Admin Panel.
  2. Select Sign in with Auth.
  3. Complete sign-in at Alfares Auth.
  4. After Auth redirects back, Shop Assistant validates your role before showing admin data.
Open Shop Assistant Admin Open Auth Admin

Required Roles

Your Auth account must have at least one of these roles:

  • global:superadmin
  • app:shop-assistant:admin

Shop Assistant does not assign roles. Role membership is owned by the authentication microservice.

Troubleshooting

You are redirected back but the admin panel stays locked

  • Confirm your Auth account has one of the required roles.
  • Sign out and sign in again after role changes so the browser receives a fresh Auth session.
  • Use an active Auth account; disabled or expired accounts are rejected by Auth validation.

You see a 403 or missing-role message

Your identity was accepted, but the account is not authorized for Shop Assistant admin. Ask an Auth administrator to assign app:shop-assistant:admin or global:superadmin.

You see a 401 or expired-session message

Your Auth session is missing or expired. Return to the admin panel and sign in through Auth again.

Do not copy access tokens from browser developer tools, logs, or API responses into documents or chats. Shop Assistant supports the Auth redirect flow so operators do not need to handle raw tokens during normal admin access.
Admin APIs still enforce server-side JWT validation and role checks. The frontend gate is a convenience layer; it is not the only protection for prompts, models, settings, operations, or agent diagnostics.
